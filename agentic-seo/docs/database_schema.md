## Schema Details

### `profiles`
Extends Supabase `auth.users`. Auto-created on signup via trigger.

```sql
id          UUID PRIMARY KEY  -- = auth.users.id
full_name   TEXT
role        TEXT CHECK IN ('admin','seo_manager','seo_executive','content_writer')
avatar_url  TEXT
created_at  TIMESTAMPTZ
updated_at  TIMESTAMPTZ
```

---

### `clients`
Each client is a company/website the agency manages.

```sql
id          UUID PK
name        TEXT NOT NULL
domain      TEXT
logo_url    TEXT
description TEXT
created_by  UUID FK → profiles
created_at  TIMESTAMPTZ
updated_at  TIMESTAMPTZ
```

---

### `client_members`
Many-to-many join table. Controls which users can access which clients.

```sql
id         UUID PK
client_id  UUID FK → clients  (CASCADE)
user_id    UUID FK → profiles (CASCADE)
created_at TIMESTAMPTZ
UNIQUE (client_id, user_id)
```

---

### `departments` ★ New (007)
The 3 fixed agency departments.

```sql
id          UUID PK
name        TEXT NOT NULL
slug        TEXT UNIQUE  -- 'seo', 'execution', 'design'
description TEXT
icon        TEXT         -- lucide icon name
is_active   BOOLEAN DEFAULT TRUE
created_at  TIMESTAMPTZ
```

**Seeded data:**
- `seo` → SEO Department
- `execution` → Execution Department  
- `design` → Design Department

---

### `department_members` ★ New (007)
A user's role within a specific department and client.

```sql
id            UUID PK
user_id       UUID FK → profiles
department_id UUID FK → departments
client_id     UUID FK → clients
dept_role     TEXT CHECK IN ('department_head','team_lead','employee','client_viewer')
created_at    TIMESTAMPTZ
UNIQUE (user_id, department_id, client_id)
```

---

### `chat_sessions`
A conversation session between a user and the Hermes agent.

```sql
id            UUID PK
client_id     UUID FK → clients
user_id       UUID FK → profiles
department_id UUID FK → departments  -- ★ Added in 007 (nullable)
title         TEXT DEFAULT 'New Session'
created_at    TIMESTAMPTZ
updated_at    TIMESTAMPTZ
```

---

### `chat_messages`
Individual messages within a chat session.

```sql
id         UUID PK
session_id UUID FK → chat_sessions
client_id  UUID FK → clients
role       TEXT CHECK IN ('user','assistant','tool')
content    TEXT
tool_name  TEXT
metadata   JSONB DEFAULT '{}'
created_at TIMESTAMPTZ
```

---

### `tasks`
Tasks generated by the AI agent or user during a session.

```sql
id            UUID PK
client_id     UUID FK → clients
department_id UUID FK → departments  -- ★ Added in 007 (nullable)
session_id    UUID FK → chat_sessions (nullable)
user_id       UUID FK → profiles (nullable)
title         TEXT
description   TEXT
status        TEXT CHECK IN ('pending','running','completed','failed')
output        JSONB DEFAULT '{}'
steps         JSONB DEFAULT '[]'
created_at    TIMESTAMPTZ
updated_at    TIMESTAMPTZ
```

---

### `approvals`
Human-in-the-loop checkpoint. Pauses workflow until decided.

```sql
id            UUID PK
client_id     UUID FK → clients
department_id UUID FK → departments  -- ★ Added in 007 (nullable)
task_id       UUID FK → tasks (nullable)
session_id    UUID FK → chat_sessions (nullable)
action_type   TEXT  -- e.g. 'publish_backlink', 'send_email'
description   TEXT
payload       JSONB DEFAULT '{}'
status        TEXT CHECK IN ('pending','approved','rejected')
decided_by    UUID FK → profiles (nullable)
decided_at    TIMESTAMPTZ
created_at    TIMESTAMPTZ
```

---

### `workflow_templates`
Reusable workflow definitions. Steps are a JSONB array of step objects.

```sql
id            UUID PK
name          TEXT
description   TEXT
steps         JSONB DEFAULT '[]'
department_id UUID FK → departments  -- ★ Added in 007 (nullable)
created_at    TIMESTAMPTZ
```

**Step object schema:**
```json
{
  "type": "hermes_task" | "browser_use_task" | "approval",
  "name": "Human-readable step name",
  "skill": "hermes_skill_name"   // only for hermes_task / browser_use_task
}
```

---

### `task_runs`
An active execution instance of a `workflow_template` for a specific client.

```sql
id                   UUID PK
client_id            UUID FK → clients
workflow_template_id UUID FK → workflow_templates
department_id        UUID FK → departments  -- ★ Added in 007 (nullable)
status               TEXT CHECK IN ('pending','running','waiting_approval','completed','failed')
current_step_index   INT DEFAULT 0
state                JSONB DEFAULT '{}'  -- runtime config (target backlinks, min DA, etc.)
created_at           TIMESTAMPTZ
updated_at           TIMESTAMPTZ
```

---

### `keywords`
Keywords tracked for a client's SEO strategy.

```sql
id           UUID PK
client_id    UUID FK → clients
keyword      TEXT
landing_page TEXT
created_at   TIMESTAMPTZ
updated_at   TIMESTAMPTZ
```

---

### `backlinks`
Individual backlinks built as part of an SEO campaign.

```sql
id          UUID PK
client_id   UUID FK → clients
keyword_id  UUID FK → keywords (nullable)
source_url  TEXT
target_url  TEXT
da          NUMERIC  -- Domain Authority
pa          NUMERIC  -- Page Authority
spam_score  NUMERIC
status      TEXT CHECK IN ('pending','submitted','approved','rejected','verified')
result_url  TEXT
created_at  TIMESTAMPTZ
updated_at  TIMESTAMPTZ
```

---

### `client_memory`
Per-client persistent storage. Used by agents to remember client-specific information.

```sql
id         UUID PK
client_id  UUID FK → clients
key        TEXT
value      JSONB
created_at TIMESTAMPTZ
updated_at TIMESTAMPTZ
UNIQUE (client_id, key)
```

---

### `agency_memory`
Global agency-wide storage. Shared across all clients.

```sql
id         UUID PK
key        TEXT UNIQUE
value      JSONB
created_at TIMESTAMPTZ
updated_at TIMESTAMPTZ
```

## RLS Policy Summary

All tables have RLS enabled. The core access pattern is:

- **Admins** see everything on all tables.
- **All other users** can only see rows where `client_id` is a client they are a member of (via `client_members`).
- **Approval decisions** additionally require `role IN ('admin', 'seo_manager')`.

> **Phase 3 Note**: Department-level RLS (filtering by `department_id` via `department_members`) is not yet implemented. All client members currently see all data for their client regardless of department.
