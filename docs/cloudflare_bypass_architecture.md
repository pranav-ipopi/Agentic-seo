# Cloudflare Bypass & Stealth Architecture

This document outlines the architectural decisions and implementations for bypassing Cloudflare Turnstile and maintaining undetectable stealth across 400+ targets in the backlink automation engine.

## 1. The 3-Tier Bypass Strategy

Cloudflare Turnstile presents significant challenges to standard Playwright automation. To maximize speed, minimize proxy bandwidth, and guarantee bypass success, we utilize a 3-Tier fallback system.

### Tier 0: The `cf_clearance` Cookie Pool (Passive Injection)
The fastest and cheapest way to bypass Cloudflare is to reuse a valid cryptographic clearance token (`cf_clearance`).
- **Mechanism**: The `ProxyManager` maintains an in-memory dictionary pool mapping `(proxy_url, domain) -> cf_clearance_cookie`. 
- **Implementation**: When `StealthBrowserManager` creates a new isolated context, it fetches all valid cookies for the active proxy from the `ProxyManager` and injects them natively into Playwright (`context.add_cookies()`). Playwright natively handles domain routing.
- **Domain Scoping & Buffer**: To prevent fingerprint bloating, injected cookies are strictly scoped to the target domain natively by Playwright. Additionally, a 300-second (5-minute) TTL buffer is applied to prevent cookies from expiring mid-session, which could trigger a hard block.
- **User-Agent Pinning**: Cloudflare strictly binds the `cf_clearance` token to the `User-Agent`. The exact User-Agent generated by the `ProxyManager` is injected into the context via Playwright options.
- **TTL**: Cookies remain valid for ~30 minutes, allowing near-instantaneous bypasses for all subsequent tasks routed to that domain on that proxy within the window.

### Tier 1: Hardware-Level Interaction (Xvfb + PyAutoGUI + CDP)
If no valid cookie exists, we drop to physical hardware automation to mimic human interactions and clear the Turnstile checkbox.
- **Virtual Display**: On headless VPS environments, `xvfbwrapper` spins up a virtual X11 framebuffer (`Xvfb`).
- **Coordinate Conversion**: Playwright bounding boxes are relative to the viewport. The engine natively extracts `window.screenX/Y` and UI chrome height to map elements perfectly to absolute PyAutoGUI screen coordinates.
- **Physical Mouse Control**: `PyAutoGUI` connects to the X11 display to perform OS-level physical clicks, circumventing Cloudflare's detection of synthetic JavaScript events (like `element.click()`).
- **Fitts's Law Kinetics**: The cursor traces a parabolic arc to the target based on Fitts's Law. It utilizes a 200ms minimum movement floor (to avoid "impossible" bot speed), randomized cognitive start delays, and 1-3px destination wiggles to perfectly simulate human muscle tremor jitter.
- **Smart Polling**: Instead of static delays, a 10-second smart polling loop dynamically checks for the destruction of the challenge iframe or the creation of the `cf_clearance` cookie, minimizing wait times.

### Tier 2: Isolated Synchronous Fallback (`sb.solve_captcha()`)
If PyAutoGUI fails to clear a strict Turnstile, we escalate to SeleniumBase's native CDP solver.
- **OS-Level Isolation**: SeleniumBase's `solve_captcha()` is deeply synchronous. To prevent deadlocking Playwright's async event loop and to guarantee cleanup, Tier 2 executes inside a strictly timed `subprocess` (35s timeout). 
- **Process Obliteration**: If SeleniumBase hangs indefinitely inside ChromeDriver, the OS-level subprocess timeout triggers, safely annihilating the entire child process tree and preventing zombie Chrome memory leaks.
- **Harvesting**: Once the isolated Chrome instance successfully solves the captcha, we extract the newly minted `cf_clearance` cookie via stdout JSON and return it to the async Playwright loop.
- **Loopback**: The cookie is injected into Playwright, stored permanently in the `ProxyManager`'s Tier 0 pool, and the target page is reloaded.

## 2. Proxy Strategy: DataImpulse Sticky Sessions

Cloudflare strictly binds the `cf_clearance` cookie to the **Exit IP** that solved the challenge. If the IP rotates, the cookie is instantly invalidated.

### IP Whitelisting & Cost Reduction
To avoid passing usernames and passwords on every request (which incurs processing overhead), the primary server/VPS IPs are whitelisted inside the DataImpulse dashboard.

### Dynamic Credential Fallback (Resiliency)
If IP whitelisting fails (e.g., propagation delays, dynamic IP shifts), the `ProxyManager` natively intercepts the connection rejection (`ERR_INVALID_AUTH_CREDENTIALS` or HTTP `407`). 
- It immediately parses `DATAIMPULSE_LOGIN` and `DATAIMPULSE_PASSWORD` from the `.env` variables.
- It dynamically reconstructs the proxy string to inject authentication (e.g., `http://user:pass@gw.dataimpulse.com:10000`).
- It seamlessly retries the connection automatically without crashing the worker or dropping the task.

### Sticky Sessions via Port Ranges
To maintain static Exit IPs while utilizing a gateway, DataImpulse provides sticky sessions mapped to specific port numbers (Ports 10000 - 20000). 
- **Dynamic Generation**: `ProxyManager.load_proxies()` contains custom parsing logic to read port ranges (e.g., `gw.dataimpulse.com:10000-10020`) from `configs/proxies.txt`.
- **The Pool**: It automatically unpacks this range into 21 discrete, concurrent proxy targets. 
- **Lifecycle**: When the engine routes traffic through Port `10000`, DataImpulse holds the Exit IP completely static for 30 minutes. This provides the exact operational window needed for the Tier 0 `cf_clearance` cookies to remain valid. After 30 minutes, DataImpulse silently rotates the IP, the old cookie naturally expires from the `ProxyManager` pool, and Tier 1 automatically acquires a new one.

## 3. Playwright Resource Optimization
To prevent Cloudflare's behavioral analytics scripts from detecting automation, we avoid blocking critical resources:
- `cdn-cgi/*` routes and `type="other"` CDP injections are explicitly **allowed** to pass through the `StealthBrowserManager` route handlers. Blocking these silences Turnstile completely, causing infinite hangs.
- Media elements (`images`, `fonts`) are currently allowed as Turnstile heavily penalizes headless browsers that drop visual rendering assets. Pure tracking analytics (e.g., Google Analytics, Hotjar) are aggressively blocked to save proxy bandwidth.

## 4. Fingerprint Spoofing & Stealth
Because `Xvfb` is a virtual framebuffer without hardware GPU passthrough, default headless profiles emit distinct non-human signatures (like `WebGL: null`). 
- **WebGL Masking**: We unconditionally inject an initialization script into every Playwright context that overrides `WebGLRenderingContext`. It intercepts `getParameter` calls and spoofs a realistic `Intel Iris OpenGL Engine` to satisfy Cloudflare's GPU checks.
- **Canvas Noise**: We apply subtle mathematical noise to `HTMLCanvasElement.prototype.toDataURL` to break static canvas hashes, ensuring Cloudflare's fingerprinting algorithms cannot correlate sessions.
